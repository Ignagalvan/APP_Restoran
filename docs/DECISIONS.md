# Decisiones — Restaurant OS

## DEC-001
Primero se desarrollará una web app responsive/PWA, no una app nativa.

**Motivo:** el comensal debe escanear un QR y usar el sistema sin instalar nada.

## DEC-002
El producto será una plataforma modular configurable.

**Motivo:** permitir que cada restaurante active o desactive módulos sin crear versiones distintas del código.

## DEC-003
Codex no decide funcionalidades.

**Motivo:** las decisiones funcionales las toma el Product Owner y se documentan antes de implementar.

## DEC-004
La primera versión se enfocará en cuenta, pago, división y feedback.

**Motivo:** resolver primero el dolor más fuerte del comensal y del restaurante.

## DEC-005
El modelo comercial será acompañado, no puramente self-service.

**Motivo:** se busca generar relación mensual, mantenimiento, mejora continua y personalización mediante configuración.
