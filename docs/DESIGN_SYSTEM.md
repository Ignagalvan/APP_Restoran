# Design System — Restaurant OS

## 1. Propósito

Este documento define la identidad visual y de experiencia de Restaurant OS. Es la fuente oficial para orientar futuras decisiones de UX, UI, animaciones y diseño, manteniendo una experiencia coherente a medida que el producto evoluciona.

## 2. Personalidad del producto

Restaurant OS debe sentirse:

- Elegante.
- Premium.
- Moderna.
- Humana.
- Intuitiva.
- Cálida.
- Rápida.
- Confiable.
- Atemporal.

No debe sentirse futurista, fría, corporativa ni infantil. La identidad debe acompañar la hospitalidad del restaurante sin competir con ella.

## 3. Objetivos del diseño

El diseño debe:

- Reducir fricción.
- Generar confianza.
- Evitar que el usuario dude sobre qué hacer.
- Guiar naturalmente cada acción.
- Hacer que la tecnología pase desapercibida.
- Potenciar la experiencia gastronómica.

La experiencia debe ser evidente y simple, sin requerir explicaciones ni aprendizaje previo.

## 4. Principios visuales

- Usar espacio en blanco generoso.
- Construir una jerarquía visual clara.
- Destacar una acción principal por pantalla.
- Mostrar como máximo tres acciones principales a la vez.
- Dar protagonismo a la tipografía.
- Utilizar iconografía simple y comprensible.
- Crear profundidad de manera sutil.
- Aplicar glassmorphism con mucha moderación y solo cuando aporte valor.
- Usar sombras suaves.
- Favorecer bordes redondeados.
- Mantener las animaciones discretas.
- Evitar el ruido visual y la sobrecarga cognitiva.

## 5. Principios de interacción

- La aplicación debe sentirse inmediata.
- Cada pantalla debe tener un único objetivo.
- Las transiciones deben reforzar la comprensión.
- Las animaciones nunca deben ser decorativas.
- Cada interacción debe aportar claridad.
- La velocidad percibida forma parte de la experiencia.
- Las acciones deben ser evidentes, cómodas y fáciles de completar.

## 6. Inspiraciones

Referencias conceptuales:

- Apple Wallet.
- Apple Human Interface Guidelines.
- Linear.
- Fintech premium.
- Hospitality premium.

> Estas referencias sirven únicamente como inspiración.

No se deben copiar interfaces ni replicar componentes. Restaurant OS debe construir y sostener una identidad propia.

## 7. Accesibilidad

- Botones grandes.
- Alto contraste.
- Excelente legibilidad.
- Uso cómodo con una mano.
- Tamaños táctiles cómodos.
- Experiencia apta para personas mayores.
- Navegación intuitiva.
- Espaciado que facilite la lectura y la interacción.

La accesibilidad forma parte de la calidad esencial del producto, no es un agregado posterior.

## 8. Qué debemos evitar

- Exceso de información.
- Muchas acciones en una misma pantalla.
- Glassmorphism exagerado.
- Interfaces futuristas.
- Colores estridentes.
- Animaciones excesivas o decorativas.
- Interfaces que requieran aprendizaje.
- Complejidad innecesaria.
- Ruido visual.
- Elementos que distraigan de la experiencia del restaurante.

## 9. Filosofía final

La tecnología debe desaparecer detrás de una experiencia cálida, confiable y sencilla. El restaurante permanece como protagonista y cada decisión de diseño debe facilitar el momento que vive el usuario.

El usuario nunca debería pensar:

> “¿Cómo funciona esta aplicación?”

Debería pensar:

> “Ya sé cómo usarla.”

Y al finalizar su experiencia:

> “Fue la forma más simple y agradable de interactuar con un restaurante.”

## 10. Patrones de interacción aprobados

- Tabs con transiciones suaves.
- Cards simples, limpias y componibles.
- Estados claros: seleccionado, no seleccionado, deshabilitado, listo y vacío.
- CTA inferior fijo cuando mejore la experiencia.
- Microanimaciones CSS de 200–300 ms.
- Componentes reutilizables, alto contraste y accesibilidad.
- Respeto obligatorio de `prefers-reduced-motion`.

No usar neumorfismo, efectos futuristas, botones metálicos, Dock, Dynamic Island, fondos animados, Framer Motion ni dependencias visuales nuevas. Restaurant OS conserva una identidad premium, minimalista, cálida, rápida y confiable.
